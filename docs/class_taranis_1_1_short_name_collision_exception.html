Detailed Description

The ShortNameCollisionException class is an exception which occures when two arguments have the same short name. So if the user defines a CLI with two arguments which both have the same starting character this exception will be thrown because both arguments will have the same short name.

CommandLineInterface::build()
.WithValue("server", "Set the server ip address.")
.WithValue("serial", "Set the serial number.");

The above would generate this exception becuase both arguments would want to use the same short name -s.

Parameters
newArgNameis the name of the argument being added which collides with a pre-existing argument.
existingArgNameis the name of the argument which already exists that is being collided with.
