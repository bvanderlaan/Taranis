6 Taranis is a toolkit which can be used in Qt projects to provide a command line interface for inputting start up parameters into your application.
7 
8 GUI applications or background processes tend to have flags now and again to allow users to alter how the application will run. They might have flags
9 to change their logging level or alter server connection addresses. The code to process the command line arguments fed into an application is largely
10 boiler plate but to do it nicely is a bit of a pain as there are lots of variants in how values can be inputted by the user.
11 
12 ***Why re-invent the wheel? Just use Taranis!***
13 
14 Taranis is a class you can instantiate which will handle parsing whatever type of command line argument your application requires. It will deal with
15 case sensitivity and differences in flag syntax such as backslashes (/), dashes (-), or double dashes (--) as well as differences in conventions such
16 as triggering help with either an *h* or a question mark (?). Taranis will also provide long and short forms of your desired flags such as *--help* or *-h*.
17 
18 ## What you get with Taranis
19 
20 When you use Taranis to process command line inputs you get all the niceties you come to expect as a user from a command line interface for free!
21 Simply define what arguments your application will support and let Taranis do the rest.
22 
23 The niceties you get with Taranis include:
24 
25 * Case insensitive flag comparisons.
26 * Flag short names, you register an argument called server and Taranis will accept server as well as s
27 * Built in version argument. Tell Taranis your applications version and it will allow users to query for it on the command line.
28 * Built in help argument. Its always nice to have help now and again and with Taranis you'll be able to print out some help text for your users describing the command line interface for your application via the help command for free.
29 
30 ## Usage
31 
32 To use Taranis you need to build a **CommandLineInterface** object which you do through the **CommandLineInterface::build()** method. The build method has a
33 fluid interface allowing you to define the command line interface your application will have by using the *With* methods.
34 
35 ### Example 1 - A simple Command Line Interface
36 ~~~{.cpp}
37  #include <QCoreApplication>
38  #include "Taranis/CommandLineInterface.hpp"
39  #include "myWidget.hpp"
40  using namespace Taranis;
41  int main(int argc, char *argv[])
42  {
43  QCoreApplication a(argc, argv);
44  MyWidget w;
45 
46  CommandLineInterface arguments = CommandLineInterface::build()
47  .WithName("My Cool App")
48  .WithVersion("1.2.3.4-abc");
49  w.show();
50  return a.exec();
51  }
52 ~~~
53 
54 With the above your application will now be sporting a command line interface which accepts two flags
55 
56 * **--version:** Will print your applications version to the terminal then exit your application.
57 * **--help:** Will print a typical help message to the terminal then exit your application.
58 Users can now query your applications version by typing:
59 
60 > $ myApp.exe --version
61 
62 or
63 > $ myApp.exe -v
64 
65 Users can also get some help text which will display your applications name and version along with the description of the version and help command line arguments.
66 > $ myApp.exe --help
67 
68 or
69 > $ myApp.exe -h
70 
71 or
72 > $ myApp.exe -?

74 ### Example 2 - A Custom Command Line Interface
75 ~~~{.cpp}
76  #include <QCoreApplication>
77  #include "Taranis/CommandLineInterface.hpp"
78  #include "myWidget.hpp"
79  using namespace Taranis;
80  int main(int argc, char *argv[])
81  {
82  QCoreApplication a(argc, argv);
83  MyWidget w;
84 
85  CommandLineInterface arguments = CommandLineInterface::build()
86  .WithName("My Cool App")
87  .WithDescription("This application is really good and has a CLI to boot!")
88  .WithVersion("1.2.3.4-abc");
89  .WithFlag("debug", "Enable debug mode")
90 
91  if( arguments["debug"].toBool() )
92  {
93  w.enableDebug();
94  }
95 
96  w.show();
97  return a.exec();
98  }
99 ~~~
100 
101 In this example you get all the same features as in Example 1 however now your application will accept the --debug flag.
102 Users can type any of the following commands to enable debug mode.
103 
104 > $ myApp.exe --Debug<br>
105 > $ myApp.exe --debug<br>
106 > $ myApp.exe -d
107 
108 Not only does your application have a debug mode flag but your flag will now be shown in the built in help message displayed when users use the *--help* argument.

110 ### Example 3 - A Custom Command Line Interface which Accepts Values
111 ~~~{.cpp}
112  #include <QCoreApplication>
113  #include <QHostAddress>
114  #include "Taranis/CommandLineInterface.hpp"
115  #include "myWidget.hpp"
116  using namespace Taranis;
117  int main(int argc, char *argv[])
118  {
119  QCoreApplication a(argc, argv);
120  MyWidget w;
121 
122  CommandLineInterface arguments = CommandLineInterface::build()
123  .WithName("My Cool App")
124  .WithVersion("1.2.3.4-abc");
125  .WithFlag("debug", "Enable debug mode.")
126  .WithValue("server", "The IP address to the server.")
127 
128  if ( arguments["debug"].toBool() ) {...}
129  if ( arguments["server"].isValid() )
130  {
131  w.setServerIp( QHostAddress( arguments["server"].toString() ) );
132  }
133 
134  w.show();
135  return a.exec();
136  }
137 ~~~
138 
139 In this example we added another argument but this one expects a value. Users can input the value in a number of ways.
140 > $ myApp.exe --server 1.2.3.4<br>
141 > $ myApp.exe --server=1.2.3.4<br>
142 > $ myApp.exe --server:1.2.3.4
143 
144 Just like with flags you also get a short form of your value argument.
145 
146 > $ myApp.exe --s 1.2.3.4<br>
147 > $ myApp.exe --s=1.2.3.4<br>
148 > $ myApp.exe --s:1.2.3.4
149 
150 Lastly the server argument also shows up in the built in help message displayed when users use the *--help* argument.
